# 회사 공유허브 UI/기능 지식 문서

## 1. 프로젝트 개요

### 앱 목적
회사 내부에서 사용하는 AI 프로그램, 공용 계정, 유료 결제 정보, 사이트 링크를 한곳에서 확인하고 관리하는 **AI 유료결제 현황판 + 공유허브** 앱을 만든다.

### 핵심 방향
- 계정 공유와 사이트 공유를 분리한다.
- 계정 공유는 메인 대시보드로 구성한다.
- 사이트 공유는 우측 세로형 Window 블럭으로 구성한다.
- AI 유료결제 현황판을 제공한다.
- 사용자가 새로운 AI 프로그램과 사이트를 직접 등록할 수 있게 한다.
- 등록된 정보는 현황판에 자동 반영된다.
- 카테고리는 자동 생성하되 사용자가 수정할 수 있게 한다.
- 수정된 카테고리는 현황판의 새로운 카테고리로 반영한다.
- 현황판은 카테고리별, 등록일 기준으로 정리해서 보여준다.
- 각 사이트 또는 서비스의 사용만료기한을 날짜 기준 그래프로 보여준다.

## 2. 브랜드 컬러 시스템

| 컬러 칩 | 변수명 | 색상 코드 | 용도 및 설명 |
|---|---|---:|---|
| ■ | `brand-main` | `#333399` | 코어 브랜드 메인 테마. 딥 바이올렛 / 울트라마린. 주요 버튼, 그라데이션 타이틀, 활성 강조색에 사용 |
| ■ | `brand-accent` | `#FF5E36` | 핵심 액센트 컬러. 비비드 코랄 오렌지. 신규 알림 배지, 액션 유도 포인트 강조에 사용 |
| ■ | `brand-bg` | `#F4F7FB` | 기본 배경색. 아주 맑고 연한 블루 계열의 웜 그레이 |
| ■ | `brand-subBg` | `#DCE4F2` | 보조 배경색. 차분한 아이스 블루. 글래스모피즘 패널 및 모달 베이스 컬러 |
| ■ | `brand-cool` | `#B3C5EA` | 쿨톤 블루. 파스텔 블루. 배경 글로우 그라데이션 포인트 및 스레드 라인 |
| ■ | `brand-warm` | `#FCE4D6` | 웜톤 베이지. 파스텔 살구. 우측 하단 글로우 및 따뜻한 카드 포인트 |

### 카드 컬러 적용 규칙
- 각 카드에는 컬러칩 기반 색상을 적용한다.
- 기본값은 지정된 컬러칩 중 랜덤으로 자동 선택한다.
- 사용자가 카드 작성/수정 시 색상을 직접 선택할 수 있게 한다.
- 카드 색상은 카드 배경, 상단 포인트 라인, 아이콘 배경, 버튼 포인트 등에 적용한다.

## 4. 배경 및 글래스모피즘 스타일

### 기본 배경
- 기본 바탕색: `#F4F7FB`
- 전체 화면에 부드러운 글로우 배경을 적용한다.

### Background Glow Effect

| 위치 | 색상 |
|---|---|
| 좌측 상단 글로우 | `rgba(179, 197, 234, 0.25)` |
| 우측 하단 글로우 | `rgba(252, 228, 214, 0.20)` |

### 배경 우측 하단 광채 포인트
- 우측 하단에 은은한 광채 효과를 둔다.
- 과하게 튀지 않게 전체 배경에 자연스럽게 섞이도록 한다.
- `brand-warm` 계열을 기반으로 부드러운 원형 blur glow를 사용한다.

### Glassmorphism Panel

| 항목 | 값 |
|---|---|
| 배경색 | `rgba(220, 228, 242, 0.45)` |
| Blur | `backdrop-filter: blur(12px)` |
| 테두리 | `rgba(255, 255, 255, 0.70)` |
| 효과 | 반투명 유리 질감, 부드러운 그림자, 은은한 입체감 |

### 메인 타이틀 그라데이션 텍스트

```css
linear-gradient(135deg, #1e1b4b 0%, #333399 100%)
```

- 어두운 네이비에서 메인 퍼플로 이어지는 세련된 그라데이션을 적용한다.
- 앱 타이틀, 현황판 주요 제목, 섹션 헤더에 사용할 수 있다.

## 6. 전체 레이아웃 구조

### 기본 화면 구성

```text
┌────────────────────────────────────────────────────────────┐
│ 회사 공유허브 / AI 유료결제 현황판                         │
├──────────────────────────────────────┬─────────────────────┤
│ 계정 공유 메인 대시보드               │ 사이트 공유 Window   │
│                                      │ 세로 블럭            │
│ [카테고리 필터] [검색] [새 AI 등록]   │ [사이트 등록]        │
│                                      │                     │
│ [AI 유료결제 현황판]                  │ [사이트 카드]        │
│                                      │ [사이트 카드]        │
│ [계정 카드] [계정 카드] [계정 카드]   │ [사이트 카드]        │

```

### 영역별 역할

| 영역 | 설명 |
|---|---|
| 좌측 메인 영역 | 계정 공유 대시보드. AI 프로그램, 유료결제 현황, 계정 정보를 중심으로 표시 |
| 우측 세로 영역 | 사이트 공유 대시보드. 참고 사이트와 링크 바로가기 버튼을 Window 블럭처럼 표시 |
| 상단 요약 영역 | 전체 AI 프로그램 수, 유료결제 총액, 만료 임박 서비스, 현재 사용자 현황 표시 |
| 그래프 영역 | 각 서비스의 사용만료기한을 날짜 기준 그래프로 표시 |

## 7. 계정 공유 카드 정보 구조

계정 공유 카드는 AI 프로그램 및 공용 계정 정보를 보여준다.

### 필수 표시 항목

| 필드 | 설명 | 예시 |
|---|---|---|
| 서비스명 | AI 서비스 또는 프로그램 이름 | ChatGPT, Kling, Runway |
| 플랜명 | 사용 중인 플랜 | Plus, Pro, Enterprise |
| 결제 내용 | 결제한 상품 또는 목적 | GPT Plus 1개월, Kling Pro 영상 생성용 |
| 결제 주기 | 월간/연간/1회성 등 | 월간, 연간 |
| 결제 금액 | 비용 | 29,000원, $20 |
| 아이디 | 로그인 ID. 공개 표시 | company_ai@gmail.com |
| 비밀번호 | 일부만 공개 | ab••••34 |
| 현재 사용자 | 현재 사용 중인 직원명 | 김OO |
| 사용기간 | 사용 시작일~만료일 | 2026.06.01~2026.07.01 |
| 카테고리 | 자동 생성 가능, 사용자 수정 가능 | 영상 생성, 문서 작성, 이미지 생성 |
| 등록일 | 최초 등록일 | 2026.06.08 |
| 색상 | 카드 테마 색상 | brand-main, brand-accent 등 |
| 아이콘/이모지 | 제목과 서비스명을 기반으로 자동 추천, 사용자 수정 가능 | 🎬, 🤖, 🎨 |

### 비밀번호 표시 규칙
- 아이디는 공개한다.
- 비밀번호는 일부만 공개한다.
- 전체 비밀번호를 카드에서 그대로 노출하지 않는다.
- 예시:

```text
비밀번호: ab••••34
비밀번호: ••••••89
```

### 비밀번호 마스킹 규칙
- 6자 이상이면 앞 2글자 + 가운데 마스킹 + 뒤 2글자 형태로 표시한다.
- 6자 미만이면 전체 또는 대부분을 마스킹한다.
- 상세 보기에서도 기본은 마스킹 상태로 표시한다.

## 8. 사이트 공유 카드 정보 구조

사이트 공유는 우측 세로형 Window 블럭에 표시한다.

### 필수 표시 항목

| 필드 | 설명 | 예시 |
|---|---|---|
| 사이트명 | 사이트 이름 | Kling, 법령정보센터 |
| 설명 | 사이트 사용 목적 | AI 영상 생성, 건축법 확인 |
| URL | 실제 이동 링크 | https://klingai.com |
| 카테고리 | 자동 생성 및 사용자 수정 가능 | AI 영상, 법규, 이미지 레퍼런스 |
| 등록일 | 최초 등록일 | 2026.06.08 |
| 색상 | 카드 테마 색상 | brand-cool, brand-warm 등 |
| 아이콘/이모지 | 제목과 사이트 내용을 기반으로 자동 추천 | 🌐, 📚, 🎬 |

### 링크 처리 규칙
- 사이트 공유 카드에는 URL을 길게 텍스트로 노출하지 않는다.
- URL이 입력되면 반드시 `바로가기` 버튼을 생성한다.
- `바로가기` 버튼을 클릭하면 입력한 사이트 주소로 이동한다.
- 새 탭으로 열리게 한다.
- `target="_blank"`와 `rel="noopener noreferrer"`를 적용한다.
- URL에 `http://` 또는 `https://`가 없으면 자동으로 `https://`를 붙인다.
- URL이 비어 있으면 `바로가기` 버튼을 숨기거나 비활성화한다.

### URL 보정 예시

| 입력값 | 실제 이동 주소 |
|---|---|
| `naver.com` | `https://naver.com` |
| `google.com` | `https://google.com` |
| `https://klingai.com` | `https://klingai.com` |

## 10. AI 유료결제 현황판

### 목적
회사에서 사용 중인 AI 유료 프로그램의 결제 상태, 사용자, 사용기간, 만료기한을 한눈에 보여준다.

### 표시 정보

| 항목 | 설명 |
|---|---|
| 전체 AI 프로그램 수 | 등록된 AI 프로그램 전체 개수 |
| 유료 결제 프로그램 수 | 결제 정보가 있는 프로그램 수 |
| 월 결제 예상 금액 | 결제 주기가 월간인 항목 합산 |
| 연 결제 예상 금액 | 결제 주기가 연간인 항목 합산 |
| 만료 임박 서비스 | 만료일이 가까운 서비스 |
| 현재 사용자 | 현재 해당 계정을 사용 중인 직원 |
| 사용기간 | 서비스 또는 계정 사용 기간 |

### 정렬 규칙
- 카테고리별로 묶어서 보여준다.
- 각 카테고리 내부는 등록일 기준으로 정리한다.
- 기본 정렬은 최신 등록일이 위로 오게 한다.

## 11. 사용만료기한 그래프

### 목적
매일 날짜 기준으로 각 사이트 또는 AI 프로그램의 사용만료기한을 시각적으로 보여준다.

### 그래프 요구사항
- 날짜 흐름에 따라 각 서비스의 만료일을 표시한다.
- 만료까지 남은 기간을 직관적으로 보여준다.
- 만료 임박 서비스는 `brand-accent` 컬러로 강조한다.
- 카테고리별로 필터링 가능하게 한다.

### 예시 표현

```text
ChatGPT Plus     |████████████░░░| D-5
Kling Pro        |████████░░░░░░░| D-12
Canva Pro        |███████████████| D-90
```

## 12. 직원 내선번호 자동 표시 기능

### 기능 목적
각 직원의 이름을 입력하면 내선번호가 자동으로 표시되도록 한다.

### 기본 동작
- 직원 이름을 입력한다.
- 등록된 직원 목록에서 이름을 검색한다.
- 일치하는 직원이 있으면 내선번호를 자동 표시한다.

### 데이터 구조 예시

```ts
type Employee = {
  id: string;
  name: string;
  extension: string;
};
```

### 표시 예시

```text
현재 사용자: 김민수
내선번호: 1234
```

### 사용자 흐름
1. 계정 카드 작성/수정 화면에서 현재 사용자 이름 입력
2. 이름과 일치하는 직원이 있으면 내선번호 자동 표시
3. 일치하는 직원이 없으면 직접 입력 가능

## 13. AI 프로그램 등록 기능

### 기능 목적
새로운 AI 프로그램을 사용자가 직접 등록할 수 있게 한다.

### 등록 필드

| 필드 | 설명 |
|---|---|
| 서비스명 | AI 프로그램 이름 |
| 플랜명 | 결제 플랜 |
| 결제 내용 | 결제 목적 및 상품 설명 |
| 결제 주기 | 월간/연간/1회성 |
| 결제 금액 | 결제 금액 |
| 아이디 | 로그인 ID |
| 비밀번호 | 마스킹 대상 |
| 현재 사용자 | 사용 중인 직원명 |
| 사용기간 | 시작일~만료일 |
| 카테고리 | 자동 생성, 사용자 수정 가능 |
| 색상 | 카드 색상 선택 |
| 이모지 | 아이콘 선택 |

### 등록 후 반영
- 새로 생성된 AI 프로그램 정보는 AI 유료결제 현황판에 자동 등록된다.
- 카테고리별 목록에도 자동 반영된다.
- 사용만료기한 그래프에도 자동 반영된다.

## 14. 카테고리 자동 생성 및 수정 기능

### 자동 생성 기준
서비스명, 제목, 설명, 결제 내용을 기반으로 AI 카테고리를 자동 추론한다.

### 카테고리 예시

| 키워드 | 자동 카테고리 |
|---|---|
| Kling, Runway, Video, 영상 | 영상 생성 |
| ChatGPT, Claude, Gemini, 문서, 요약 | 문서/대화 AI |
| Midjourney, Canva, 이미지, 디자인 | 이미지/디자인 |
| Suno, 음악, 음원 | 음악 생성 |
| 법령, 건축법, 자료 | 법규/자료 |
| 기타 | 기타 |

### 사용자 수정
- 자동 생성된 카테고리는 사용자가 수정할 수 있다.
- 사용자가 새로운 카테고리를 입력하면 현황판에 새로운 카테고리로 등록된다.
- 이후 같은 카테고리를 선택할 수 있도록 옵션에 추가한다.

## 15. 카드 입체감 스타일

각 카드별 입체감을 더한다.

### 기본 카드 스타일
- 둥근 모서리: `border-radius: 20px`
- 부드러운 그림자: `box-shadow: 0 16px 40px rgba(30, 27, 75, 0.10)`
- hover 시 살짝 위로 이동: `transform: translateY(-3px)`
- hover 시 그림자 강화
- 글래스모피즘 패널 위에 카드가 떠 있는 느낌을 준다.

### 카드 내부 요소
- 상단에 색상 포인트 라인 또는 그라데이션 바를 둔다.
- 아이콘은 원형 또는 라운드 사각 배지 형태로 표시한다.
- 카드 배경은 선택된 컬러를 아주 연하게 반영한다.
- 주요 액션 버튼은 `brand-main` 또는 카드 선택 색상 기반으로 표시한다.

## 16. Supabase 데이터 구조 제안

기존 `public.memos` 테이블을 확장해서 사용할 수 있다.

### 기존 컬럼

| 컬럼 | 설명 |
|---|---|
| `id` | 고유 ID |
| `type` | `account` 또는 `site` |
| `title` | 제목 |
| `content` | 설명 또는 메모 |

|---|---|
| `emoji` | 카드 이모지 |
| `color` | 카드 색상 |
| `service_name` | 서비스명 |
| `plan_name` | 플랜명 |
| `payment_content` | 결제 내용 |
| `billing_cycle` | 결제 주기 |
| `payment_amount` | 결제 금액 |
| `login_id` | 로그인 아이디 |

### SQL 예시

```sql
alter table public.memos
add column if not exists emoji text default '🔗',
add column if not exists color text default 'brand-main',

add column if not exists plan_name text,
add column if not exists payment_content text,
add column if not exists billing_cycle text,
add column if not exists payment_amount text,
add column if not exists login_id text,
add column if not exists password text,
add column if not exists site_url text,
add column if not exists current_user text,
add column if not exists extension_number text,
add column if not exists usage_start_date date,
add column if not exists usage_end_date date,
add column if not exists category text;
```
